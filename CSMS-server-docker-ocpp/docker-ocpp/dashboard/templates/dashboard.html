Technical Methodology & Data Processing

query_stats

1. Load Forecasting Engine

Input: Historical MeterValues (kWh)

Sistem menggunakan data Time-Series dari tabel transaksi. Data dibersihkan (denoising) dan diproses menggunakan algoritma sliding window. Model memprediksi beban grid dengan mempertimbangkan tren harian (jam sibuk) dan musiman.

monitor_heart

2. Anomaly Detection

Input: ErrorCodes, Heartbeat Logs

Menggunakan algoritma Isolation Forest (Unsupervised Learning) untuk mendeteksi outlier. Jika latensi heartbeat atau frekuensi error melebihi threshold deviasi standar, sistem menurunkan Health Score secara real-time.

calendar_month

3. Probabilistic Availability

Input: Session Start/Stop Times

Menghitung probabilitas slot kosong menggunakan distribusi statistik (Bayesian Inference) berdasarkan hari dalam seminggu. Ini membantu user memilih waktu charging yang optimal dan membantu operator dalam load balancing.
